# Seat_Reservation 与 V8 登录功能整合说明

## 整合概述

本次整合将 V8 版本的滑块验证功能成功集成到 Seat_Reservation 文件中，使得系统能够自动处理滑块验证码，获取 token 并成功登录。

## 主要改进

### 1. 滑块验证功能升级
- **多重缺口检测算法**：结合边缘检测、阴影分析和纹理特征
- **真实人类轨迹模拟**：包含停顿、抖动、变速等自然行为
- **Y轴随机移动**：模拟真实滑动时的上下偏移
- **多重尝试机制**：最多尝试10次，提高成功率

### 2. 核心功能整合
- 自动检测是否需要滑块验证
- 自动获取验证码图片
- 自动识别缺口位置
- 自动生成滑动轨迹
- 自动验证并获取 token

## 文件变更

### 新增文件
- `get_bearer_token_v8.py` - 整合版本登录模块

### 修改文件
- `Seat_Reservation.py` - 更新导入为 v8 版本
  ```python
  # 原来的导入
  from get_bearer_token import get_bearer_token
  
  # 更新后的导入
  from get_bearer_token_v8 import get_bearer_token
  ```

## 使用方法

### 1. 配置账号信息
编辑 `config.yml` 文件：
```yaml
USERNAME: "你的学号"
PASSWORD: "你的密码"
CLASSROOMS_NAME: ["教学楼名称"]
USER_SEAT_ID: ["座位号"]
DATE: "today"
HOUR: 9      # 预约小时
MINUTES: 0   # 预约分钟
```

### 2. 运行程序
```bash
python Seat_Reservation.py
```

### 3. 程序流程
1. 读取配置文件
2. 等待到达预约时间
3. 调用 `get_bearer_token()` 获取登录 token
4. 自动处理滑块验证（如果需要）
5. 登录成功后开始抢座
6. 成功预约后发送通知

## 滑块验证流程

1. **获取验证码**
   - 访问验证码接口
   - 获取背景图和滑块图

2. **识别缺口**
   - 使用边缘检测算法分析图片
   - 计算缺口位置
   - 生成候选移动距离

3. **生成轨迹**
   - 模拟真实滑动行为
   - 包含加速、匀速、减速阶段
   - 添加随机抖动

4. **验证提交**
   - 加密滑动数据
   - 提交验证请求
   - 获取验证结果

## 注意事项

1. **依赖安装**
   - 确保安装了所有必要的依赖包
   - 特别是 `opencv-python` 和 `cryptography`

2. **网络环境**
   - 确保网络通畅
   - 如果验证失败，程序会自动重试

3. **成功提升**
   - V8 版本相比原版大幅提升了滑块验证成功率
   - 遇到验证失败时会自动重试最多10次

## 故障排除

